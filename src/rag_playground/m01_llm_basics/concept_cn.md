# 模块 01：大模型基础

## 你将学到

- 如何通过 Ollama 与**本地大模型**对话
- **流式输出**和**非流式输出**的区别
- **Temperature** 如何控制随机性与确定性
- 基础**提示工程**：系统提示词、少样本提示、思维链

## 为什么本地大模型很重要

云端 API（OpenAI、Anthropic）虽然方便，但存在以下问题：
- 按 Token 收费
- 将你的数据发送到外部服务器
- 需要网络连接
- 有速率限制和使用政策

通过 Ollama 运行的本地模型是免费、私密、可离线且无限制的。对于学习 RAG 来说，本地模型意味着你可以尽情实验，无需担心 API 账单。

## 对话补全的工作原理

```
用户："什么是机器学习？"
  → 系统提示词（可选）："你是一个乐于助人的助手。"
  → 模型处理 Token
  → 助手："机器学习是人工智能的一个领域……"
```

### 非流式（chat）

```python
response = await provider.chat(model, messages)
# 一次性返回完整响应
```

### 流式（chat_stream）

```python
async for token in provider.chat_stream(model, messages):
    print(token, end="")  # Token 逐个出现
```

流式输出给用户即时反馈——对于聊天应用的良好用户体验至关重要。

### Temperature（温度）

| Temperature | 行为 | 适用场景 |
|-------------|------|----------|
| 0.0 | 确定性输出，选择最可能的 Token | 事实问答、代码生成 |
| 0.5 | 平衡 | 通用对话 |
| 1.0 | 创造性、多样化输出 | 头脑风暴、创意写作 |

Temperature 越低 = 输出越聚焦，幻觉越少（对 RAG 有利）。

### 提示工程

**系统提示词**设定助手的行为方式：
```
System："仅使用提供的上下文回答问题。如果上下文中没有答案，
        请说'我没有足够的信息'。"
```

**少样本提示**提供示例：
```
User："分类：'我很喜欢这个产品'"
Assistant："正面"
User："分类：'糟糕的体验'"
Assistant："负面"
User："分类：'它还行'"
Assistant："中性"
```

**思维链**要求模型逐步推理：
```
"让我们一步步思考。首先……"
```

## 常见陷阱

1. **模型必须先拉取**：运行 `python scripts/pull_models.py`
2. **内存使用**：Qwen 7B 需要约 5-7GB 内存。确保 Ollama 有足够资源。
3. **上下文窗口**：Qwen 2.5 7B 支持 32K Token。长对话需要上下文管理。
4. **Temperature 0 ≠ 永远相同的答案**：本地模型在 temperature 0 时仍可能有微小变化。

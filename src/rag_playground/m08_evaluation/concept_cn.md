# 模块 08：评估 RAG 质量

## 你将学到

- **检索指标**：Precision@K、Recall@K、MRR、NDCG
- **生成指标**：忠实度和相关性评分
- 如何构建问答对的**测试集**
- 如何定量**比较**检索策略

## 为什么评估很重要

"你无法改进你没有衡量的东西。"RAG 系统有两种不同的失败模式：

1. **检索失败**：找不到正确的文档
2. **生成失败**：找到了正确的文档，但答案错误

评估分别衡量这两者。

## 检索指标

给定查询、相关文档 ID 集合（标准答案）和检索结果：

### Precision@K
检索到的文档中有多少是相关的？
```
Precision@3 = (top-3 中的相关数) / 3
```

### Recall@K
所有相关文档中有多少被检索到了？
```
Recall@3 = (top-3 中的相关数) / (总相关文档数)
```

### 平均倒数排名（MRR）
第一个相关结果出现在什么位置？
```
MRR = 1 / (第一个相关结果的排名)
```
如果第一个相关结果在第 2 位：MRR = 1/2 = 0.5

### 归一化折损累计增益（NDCG）
奖励排名靠前的高相关度结果：
```
DCG@K = sum(relevance_i / log2(i+1)) for i=1..K
NDCG = DCG / ideal_DCG
```

## 生成指标

精确匹配对评估生成文本没有用。取而代之地使用：

### 忠实度
生成的答案是否得到检索上下文的支持，还是产生了幻觉？
LLM 作为评判："这个答案是否仅包含存在于上下文中的信息？"

### 相关性
答案是否实际回答了提出的问题？
LLM 作为评判："这个答案是否直接回答了用户的问题？"

## 你将练习什么

1. 加载问答对的测试集
2. 运行检索评估（Precision@3、Recall@3、MRR、NDCG）
3. 运行生成评估（忠实度、相关性）
4. 在同一测试集上比较检索策略

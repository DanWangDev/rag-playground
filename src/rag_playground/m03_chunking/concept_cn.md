# 模块 03：文本切分

## 你将学到

- 为什么 RAG 系统**必须**将文档切分为片段
- 三种切分策略：**字符切分**、**递归切分**和**基于 Token 的切分**
- **片段重叠**如何提升检索质量
- 如何根据嵌入模型和大模型选择合适的片段大小
- 大小片段之间的权衡

## 为什么文本切分很重要

大模型有一个**上下文窗口**——一次能处理的最大 Token 数量。现代模型可以处理 8K-128K Token，但检索增强生成在处理**聚焦的、相关的上下文**而非整篇文档时效果最好。

如果你为一句简短的问题嵌入并检索了一整篇 10 页的文档，你就是在浪费 Token、稀释相关性并增加成本。切分将文档分解为更小的、自包含的单元：

```
文档（5000 字符）
  ├── 片段 1（500 字符）："机器学习是……"
  ├── 片段 2（500 字符）："……监督学习使用标注数据……"
  ├── 片段 3（500 字符）："……无监督学习发现模式……"
  └── 片段 4（500 字符）："……深度学习使用神经网络……"
```

每个片段获得自己的嵌入向量。当用户询问"监督学习"时，只有片段 2-3 被检索——而非整篇文档。

## 切分的工作原理

### 字符切分

最简单的策略：每 N 个字符切分一次文本。

```
chunk_size = 100
text = "AAAAABBBBBCCCCCDDDDD"  （20 字符）

片段 1："AAAAA"      （字符 0-9，10 字符）
片段 2："BBBBB"      （字符 10-19，10 字符）
```

**优点：** 简单、快速、确定性。
**缺点：** 可能在单词和句子中间切断。一个片段可能以"敏捷的棕色"结尾，下一个以"狐狸跳过……"开头。

### 递归字符切分

使用**分隔符优先级**先在自然边界上切分：

```
先尝试 "\n\n"（段落）
  → 再尝试 "\n"（行）
    → 再尝试 "。"（句子）
      → 再尝试 " "（单词）
        → 最后回退到字符切分
```

这保留了语义边界。片段将尽可能包含完整段落，不行则完整句子，仅在最后手段才在单词中间切分。

**优点：** 每个片段语义更连贯。
**缺点：** 略微复杂。片段大小可能不均匀。

### 基于 Token 的切分

字符 ≠ Token。在英语中，一个 Token 大约是 4 个字符，但也可以是一个标点符号（1 个字符）或一个长单词（10+ 个字符）。不同语言的比率也不同。

基于 Token 的切分（近似地）计算 Token 而非字符，确保片段适合模型的上下文窗口。

```
chunk_tokens = 128
text = "Transformer 架构……"（50 tokens）

→ 一个约 50 tokens 的片段（适合 128 的限制）
```

**优点：** 模型感知的大小控制。防止上下文溢出。
**缺点：** 是近似值（没有依赖 tokenizer）。因模型而异。

### 片段重叠

重叠创建滑动窗口：每个片段与相邻片段共享部分文本。

```
chunk_size = 100, overlap = 20
text = "AAAAABBBBBCCCCCDDDDDEEEEE"

片段 1："AAAAABBBBBCCCCC"  （100 字符）
片段 2："BBBBBCCCCCDDDDD"  （100 字符，与片段 1 共享 "BBBBBCCCCC"）
片段 3："CCCCCDDDDDEEEEE"  （100 字符，与片段 2 共享 "CCCCCDDDDD"）
```

**为什么重叠很重要：** 一个关键事实可能跨越两个不重叠片段的边界。如果"机器学习由 Arthur Samuel 于 1959 年发明"被分成两个片段：

```
片段 1："……机器学习由"
片段 2："Arthur Samuel 于 1959 年发明……"
```

单独看，两个片段都无法回答"谁发明了机器学习？"。有了重叠，一个片段就能捕获完整的句子。

## 选择片段大小

| 片段大小 | 最适合 | 权衡 |
|---------|-------|------|
| 小（128-256） | 事实问答、短回答 | 更多片段，更多嵌入，更多存储 |
| 中（500-1000） | 通用 RAG（默认） | 平衡的检索粒度 |
| 大（2000-4000） | 摘要、长篇回答 | 更少片段，更广上下文，检索精度较低 |

## 常见陷阱

1. **单词中间切分**：在第 73 个字符处切分可能将 "transformer" 切成 "trans" + "former"。使用单词级分隔符的递归切分可以防止这个问题。
2. **Token ≠ 字符**：500 字符 ≈ 125 Token（英语），但在日语中约 250 Token。多语言应用需要感知 Token 的切分。
3. **元数据丢失**：每个片段继承父文档的元数据。切分时不要丢失来源信息。
4. **最小片段大小**：非常小的片段（< 50 字符）通常缺乏有用的上下文。应过滤掉。

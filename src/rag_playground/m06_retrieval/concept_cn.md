# 模块 06：检索策略

## 你将学到

- **基础检索**：嵌入查询 → 搜索向量存储 → 返回 top-k
- **查询预处理**：扩展、分解和 HyDE 查询以提升召回率
- **多查询检索**：生成变体、并行搜索、融合结果
- **LLM 重排序**：使用大模型本身评分和重新排序检索到的候选项

## 为什么检索策略很重要

这是 RAG 中的"R"。朴素的检索可能因为以下原因遗漏相关文档：
- 查询使用的词汇与文档不同（词汇不匹配）
- 查询太宽泛或太窄
- 多个相关事实分散在不同文档中

高级检索策略针对每种失败模式都有应对方案。

## 策略

### 基础检索
嵌入查询，搜索向量存储，返回 top-k。简单快速。

### 查询扩展
使用 LLM 用同义词和相关术语扩展查询：
```
"ML" → "机器学习，人工智能，深度学习，神经网络"
```

### 查询分解
将复杂问题拆分为更简单的子问题：
```
"比较机器学习和量子计算" →
  Q1："什么是机器学习？"
  Q2："什么是量子计算？"
  Q3："机器学习和量子计算如何关联？"
```

### HyDE（假设文档嵌入）
生成一个假设性答案，然后嵌入_这个答案_作为查询：
```
Q："气候变化的原因是什么？"
假设："气候变化主要由温室气体排放引起……"
嵌入这个假设 → 搜索与假设相似的真实文档
```

### 多查询检索
生成 N 个查询变体，分别检索，用倒数排名融合（RRF）合并结果：
```
RRF(score) = 对所有查询求和：1 / (k + rank)
```

### LLM 重排序
检索 top-N 候选项（如 20 个），然后让 LLM 评分其相关性：
```
"从 1-10 打分，这段文字与查询'……'的相关性如何？"
```

## 常见陷阱
- 多查询更慢（N 次 API 调用）但召回率更好
- 重排序增加延迟但提高精确度
- HyDE 对事实类查询效果好，但对创意类查询效果差
- 查询扩展有时会引入噪音——测试不同的扩展策略
